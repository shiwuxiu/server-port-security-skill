# 敏感端口详细说明

## 🔴 严重风险端口

### Elasticsearch (9200/9300)
- **攻击类型**: 勒索攻击 (Meow Attack)、数据删除
- **攻击特征**: 创建 `read_me` 勒索索引，删除原有数据
- **历史案例**: 2020年大规模自动化扫描攻击，数万ES实例被勒索
- **修复方案**: 端口绑定 `127.0.0.1`，启用 X-Pack Security

### Redis (6379/6380)
- **攻击类型**: 挖矿木马、SSH公钥写入、数据篡改
- **攻击特征**: 通过未授权访问写入恶意脚本，植入挖矿程序
- **历史案例**: 众多服务器因Redis裸奔被植入挖矿木马
- **修复方案**: 端口绑定 `127.0.0.1` + 设置强密码

### MongoDB (27017)
- **攻击类型**: 勒索攻击、数据删除
- **攻击特征**: 自动化勒索脚本删除数据，留下勒索信息
- **历史案例**: 2017年大规模MongoDB勒索攻击
- **修复方案**: 启用认证，端口绑定本地

### Docker Daemon (2375/2376)
- **攻击类型**: 容器逃逸、完全接管宿主机
- **攻击特征**: 通过Docker API创建特权容器，获取宿主机root权限
- **风险等级**: **最危险**，可导致完全系统接管
- **修复方案**: 绝不暴露公网，启用TLS认证

## 🟠 高风险端口

### etcd (2379/2380)
- **用途**: 分布式配置存储，Kubernetes/Milvus依赖
- **攻击类型**: 数据篡改、配置注入
- **影响**: 可导致Milvus崩溃，Kubernetes集群被接管
- **修复方案**: 端口绑定 `127.0.0.1`

### Milvus (19530)
- **用途**: 向量数据库，AI应用核心
- **攻击类型**: 向量数据泄露、数据删除
- **修复方案**: 端口绑定 `127.0.0.1`

### Memcached (11211)
- **攻击类型**: DDoS放大攻击
- **攻击特征**: UDP反射攻击，放大倍数可达50000倍
- **历史案例**: 2018年GitHub遭受1.35Tbps DDoS攻击
- **修复方案**: 禁用UDP或绑定本地

### Hadoop YARN (8088)
- **攻击类型**: RCE远程命令执行
- **攻击特征**: 挖矿团伙首选目标
- **修复方案**: 绝不暴露公网

### Zookeeper (2181)
- **用途**: 分布式协调服务
- **攻击类型**: 配置泄露、服务劫持
- **修复方案**: 端口绑定本地 + ACL配置

## 🟡 中等风险端口

### MySQL (3306)
- **攻击类型**: 弱口令爆破、SQL注入
- **修复方案**: 强密码 + 端口绑定本地

### PostgreSQL (5432)
- **攻击类型**: 弱口令爆破
- **修复方案**: 强密码 + 端口绑定本地

### MinIO (9000/9001)
- **用途**: 对象存储
- **攻击类型**: 数据泄露
- **注意**: Console端口(9001)绝对不要暴露公网
- **修复方案**: API端口谨慎开放，Console绑定本地

### RabbitMQ (5672/15672)
- **用途**: 消息队列
- **风险**: 默认账号 guest/guest
- **修复方案**: 删除默认账号 + 端口绑定本地

### Kafka (9092)
- **用途**: 消息队列
- **攻击类型**: 数据泄露
- **修复方案**: 端口绑定本地 + 启用SASL认证
